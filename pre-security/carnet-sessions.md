### Ce que j'ai compris
La cybersécurité est divisée en deux grandes parties, la partie
offensif (attaquer, tester les systèmes) et défensif 
(protéger, surveiller, répondre aux incidents). Chaque partie 
regroupe des métiers bien distincts.

Un réseau c'est un ensemble de machines connectées entre elles 
pour communiquer. Internet est un réseau de réseaux à l'échelle 
mondiale.

Le DNS (Domain Name System) fonctionne comme un annuaire, il 
traduit un nom de domaine (exemple : google.com) en adresse IP. 
Il existe 3 types de serveurs DNS : Recursive, Root et 
Authoritative.

Une URL HTTP est composée de plusieurs parties, le protocole, 
le nom de domaine, le chemin, et les paramètres. Chaque réponse 
HTTP a un code de statut (exemple : 200 = OK, 404 = introuvable, 
500 = erreur serveur, etc.).

### Ce qui était fluide
La logique générale de la cybersécurité et la distinction 
offensif/défensif. La notion de réseau et d'internet était 
intuitive une fois les analogies bien comprises.

qu'ils transportent des informations mais leur mécanique précise 
reste encore floue.

Les DNS Record Types, je sais qu'ils existent mais je n'ai pas 
encore bien compris à quoi chaque type correspond concrètement (a assimiler encore).

La composition d'une adresse IPv6 (lettres + chiffres), 
la logique derrière ce format n'est pas encore claire même si on connait la raison de sa création.
